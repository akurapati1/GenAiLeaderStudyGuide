DOMAIN 1 — Fundamentals (~30%)
🤖 AI vs ML vs Deep Learning vs Generative AI
▼
Core Distinctions
  • AI — Broad field: machines that simulate human intelligence (reasoning, learning, problem-solving).
  • Machine Learning — Subset of AI; systems learn from data patterns without explicit programming.
  • Deep Learning — Subset of ML using neural networks with many layers; excels at images, speech, text.
  • Generative AI — Subset of DL; models that create new content (text, images, audio, code) rather than just classify or predict.
  • Key Differentiator — Traditional ML predicts; GenAI generates. GenAI learns the distribution of training data and samples from it.
📝 Exam-style Q

A company wants a system that generates product descriptions from structured specs. Which AI capability is MOST appropriate? → Generative AI / LLM

🏗️ Foundation Models & Large Language Models (LLMs)
▼
What You Must Know
  • Foundation Model — A large pre-trained model on vast data; adaptable to many tasks via fine-tuning or prompting.
  • LLM — Foundation model specifically for language; uses transformer architecture; trained via self-supervised learning.
  • Transformer Architecture — Self-attention mechanism processes entire sequences at once; enables understanding context across long inputs.
  • Tokens — Text is split into tokens (≈ word fragments). Token limits = context window size.
  • Embeddings — Numerical vector representations of text; capture semantic meaning; enable similarity search.
  • Multimodal Models — Handle multiple input types: text, images, audio, video (e.g., Gemini). Uni-modal = one type.
  • Limitations — Hallucinations (fabricated facts), bias (from training data), knowledge cutoff (no real-time data), context window limits.
📝 Exam-style Q

An LLM generates confident but factually incorrect information. This is called: → Hallucination

📊 ML Lifecycle & Data Types
▼
ML Lifecycle Stages
  • 1. Problem Definition — Define the business objective and success metrics.
  • 2. Data Collection & Preparation — Gather, clean, label data; split into train/validation/test sets.
  • 3. Model Training — Feed data into algorithm; model learns parameters to minimize loss.
  • 4. Model Evaluation — Use metrics (accuracy, BLEU, ROUGE, perplexity) on held-out test data.
  • 5. Deployment — Serve model via API or application (Vertex AI Prediction).
  • 6. Monitoring — Track performance, detect drift, retrain as needed.
Data Types
  • Structured — Rows & columns (SQL tables, spreadsheets). Easy to query.
  • Unstructured — Text, images, audio, video, PDFs. GenAI excels here.
  • Semi-structured — JSON, XML — has some organization but not rigid schema.
⚖️ Responsible AI Fundamentals
▼
Google's 7 Responsible AI Principles
  • Socially Beneficial — AI should benefit society at large.
  • Avoid Unfair Bias — Prevent discrimination; test for fairness across demographic groups.
  • Safety — Test rigorously; operate within appropriate constraints.
  • Accountable — Humans should be able to oversee and correct AI systems.
  • Privacy — Respect data privacy; obtain appropriate consent.
  • Uphold Scientific Excellence — Apply rigorous scientific standards.
  • Made Available for Uses That Accord with These Principles — Don't enable harmful applications.
📝 Exam-style Q

An AI model for loan approvals shows bias against a protected group. Which responsible AI principle is MOST violated? → Fairness / Avoid Unfair Bias

🔍 Grounding vs. RAG
▼
  • Grounding — Connecting model responses to authoritative external data sources to reduce hallucinations. The model cites or draws from real data.
  • RAG (Retrieval-Augmented Generation) — A specific grounding technique: (1) Retrieve relevant documents from a vector store; (2) Add them to the prompt context; (3) Model generates response based on retrieved docs.
  • When to use RAG — Large knowledge bases, real-time data needs, proprietary/internal docs, reduce hallucination on facts.
  • Grounding with Google Search — Vertex AI allows grounding model responses with live Google Search results.
  • Vector DB for RAG — Documents converted to embeddings; stored in Vertex AI Vector Search; semantically retrieved at query time.
📝 Exam-style Q

A company wants a chatbot to answer HR policy questions using internal policy documents without retraining the model. Best approach? → RAG (Retrieval-Augmented Generation)

🎛️ Model Parameters
▼

TemperatureRandomness / creativityMore deterministic, focusedMore creative, varied, risky
Top-p (nucleus)Pool of tokens to sample fromNarrow, predictableWider vocabulary

Key Rules
  • Use low temperature (0–0.2) for: factual Q&A, code generation, structured outputs.
  • Use high temperature (0.7–1.0) for: creative writing, brainstorming, marketing copy.
  • Temperature = 0 → Greedy decoding; always picks most likely token. Fully deterministic.
📝 Exam-style Q

A chatbot generating creative story variations produces repetitive outputs. Which parameter should be increased? → Temperature

DOMAIN 4 — Business Strategies (~15%)
🏛️ SAIF — Google's Secure AI Framework
▼
6 Core Elements of SAIF
  • Expand strong security foundations to the AI ecosystem — Apply existing Google Cloud security controls (IAM, VPC) to AI.
  • Extend detection and response — Monitor AI systems for unusual behavior; include in SIEM/SOAR workflows.
  • Automate defenses — Use AI to defend AI systems; automated threat detection.
  • Harmonize platform-level controls — Consistent security controls across the AI lifecycle.
  • Adapt controls to address AI-specific risks — Defend against prompt injection, model poisoning, data exfiltration via AI.
  • Contextualize AI system risks — Evaluate risk based on deployment context and data sensitivity.
🎯 Model Selection Criteria
▼
  • Modality — What data types does your task require? Text-only, image+text, audio, video, multimodal?
  • Context Window — How much input does the model accept? Long docs need large context windows.
  • Performance / Quality — Accuracy benchmarks; evaluated on your specific task.
  • Latency — Real-time apps (chatbots) need fast inference; batch jobs can tolerate delays.
  • Cost — Larger models cost more per token. Flash models cheaper than Pro/Ultra.
  • Data Residency / Compliance — Regulated industries (healthcare, finance) may require data to stay in specific regions.
  • Open-source vs. Proprietary — Open-source (Llama) = customizable but self-managed; proprietary (Gemini) = managed, enterprise support.
📝 Exam-style Q

A healthcare org needs to summarize patient records but data cannot leave the EU. Which factor is MOST critical in model selection? → Data Residency / Compliance

📈 GenAI Adoption Strategy
▼
  • Problem-First Approach — Google recommends: START with the business problem, not the technology. Define desired outcomes first.
  • Top-Down Strategy — Leadership mandates AI adoption; top-down governance, investment, and policy.
  • Bottom-Up Strategy — Grassroots adoption; individual teams experiment and champion; then scale.
  • Needs Assessment — Identify pain points, data readiness, skill gaps, infrastructure requirements.
  • ROI Framework — Measure: time saved, cost reduction, revenue enabled, error rates reduced.
  • Organizational Readiness — Data governance, change management, training, cultural buy-in.
  • Humans in the Loop (HITL) — For high-stakes decisions, ensure humans review AI outputs. Especially critical in healthcare, legal, finance.

High-Frequency Concepts Master These

The concepts most likely to appear on the exam. Know these cold.

🌡️ Temperature

Controls output randomness. Low (0–0.2) = deterministic/factual. High (0.7–1.0) = creative/varied. Temperature = 0 → greedy decoding.

👻 Hallucination

Model generates confident but factually incorrect information. Mitigation: RAG, grounding, temperature reduction, HITL review.

🔍 RAG

Retrieval-Augmented Generation: retrieve relevant docs → inject into prompt context → generate grounded response. Reduces hallucinations without retraining.

📐 Embeddings

Numerical vector representations of text/images. Similar meanings → nearby vectors. Power semantic search and RAG pipelines.

🪟 Context Window

Maximum tokens a model can process at once (input + output). Larger = can handle longer documents. Critical for document summarization tasks.

🤖 AI Agent

System that perceives environment, makes decisions, and takes actions to achieve a goal. Can use tools (APIs, search, code execution). LLM is the "brain."

⚖️ Responsible AI

Fairness, transparency, privacy, accountability, safety, social benefit. Google's 7 principles guide ethical AI development and deployment.

🎯 Fine-tuning

Further train a pre-trained model on domain-specific data. Adapts model's weights. More expensive than prompting/RAG but enables deep style/domain adaptation.

🔗 Chain-of-Thought

Prompt model to reason step-by-step ("Let's think step by step"). Dramatically improves accuracy on reasoning, math, and multi-step tasks.

🛡️ SAIF

Google's Secure AI Framework. 6 elements for AI security: expand foundations, extend detection, automate, harmonize controls, adapt for AI risks, contextualize.

👥 HITL

Humans in the Loop — humans review/approve AI outputs before action. Critical for high-stakes domains: medical diagnosis, legal decisions, financial approvals.

📏 Eval Metrics

BLEU (machine translation quality), ROUGE (summarization recall), Perplexity (language model confidence), FID (image generation quality), Human eval.

Exam Day Strategy Pass First Try

Tactical tips for navigating the actual exam with confidence.

✅ Question Strategy
  • Read the ENTIRE question before looking at options.
  • Identify: What is the CORE problem? What constraints exist?
  • Match the scenario to a Google tool or AI technique.
  • Eliminate clearly wrong answers first (usually 2 can be cut immediately).
  • When 2 answers look right, pick the one that's most Google Cloud-specific.
  • For "most critical" questions — think about the primary blocker or constraint.
⚠️ Common Mistakes to Avoid
  • Confusing Google AI Studio (prototype) with Vertex AI Studio (production).
  • Choosing fine-tuning when RAG is cheaper and sufficient.
  • Ignoring compliance/data residency for healthcare/finance scenarios.
  • Not reading "select ALL that apply" carefully — may need multiple answers.
  • Forgetting responsible AI: always consider fairness and privacy angles.
  • Confusing Document AI (structured extraction) with Vision API (general image).
